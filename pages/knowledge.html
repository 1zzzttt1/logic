Understanding Tokens: The DNA of Large Language Models

探究模型是如何将人类语言转化为数学序列的，以及为什么 Token 计数对成本和性能至关重要。

什么是 Token？

在自然语言处理（NLP）的世界中，Token 是最小的语义单位。对于人类来说，我们阅读的是单词和句子；而对于大型语言模型（LLM）如 GPT-4 来说，它们处理的是被称为 "Tokens" 的数字块。

一个 Token 不一定是一个完整的单词。它可以是一个单词、一个单词的一部分（如前缀或后缀），甚至是标点符号。这种精细的拆分方式让模型能够更高效地处理罕见词汇。

💡 专家提示

作为通用规则，在英文文本中，1000 个 Token 大约等于 750 个单词。而在中文中，通常一个汉字对应 1 到 2 个 Token，具体取决于模型的分词器。

分词过程 (Tokenization)

分词是将原始文本转换为 Token 序列的过程。目前的 SOTA 模型多采用 Byte-Pair Encoding (BPE) 算法。

  • 1 标准化： 清理文本，处理空格和特殊字符。
  • 2 拆分： 根据预定义的词表（Vocabulary）将文本切碎。
  • 3 编码： 将每个 Token 映射到其在词表中的索引 ID。

为什么这很重要？

上下文窗口限制

每个模型都有最大的 Token 容纳量。如果你理解了 Token 的计算，你就能写出更简洁、更高效的 Prompt。

推理成本计算

大多数 API 提供商（如 OpenAI, Anthropic）都是基于 Token 数量收费的。优化 Token 使用直接等于节省成本。
