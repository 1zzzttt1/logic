# 个人 AI 知识博客 - 产品规格文档 (最终版)

*本规格文档已完成小范围统一修正，统一为 MVP 阶段的一级分类方案*

---

## 1. 产品定位

**一句话定位**：一个记录 AI 学习笔记、分享技术见解的个人知识博客网站。

**核心理念**：知识沉淀 + 公开分享，双轨并行。

---

## 2. 这个产品是什么

- 一个**静态内容网站**，用于展示和分享 AI 相关知识
- 一个**个人知识管理工具**，帮助作者整理和结构化学习内容
- 一个**技术博客**，面向对 AI 和 Claude Code 感兴趣的开发者

---

## 3. 这个产品不是什么

| 不是 | 解释 |
|------|------|
| 社交平台 | 没有用户评论、点赞、关注等社交功能 |
| 动态应用 | 不需要后端数据库，静态部署即可 |
| 多用户系统 | 仅供单人维护和发布内容 |
| 在线编辑器 | 不提供在线写文章的功能，需通过本地文件发布 |
| 商业产品 | 不涉及付费、广告、订阅等商业模式 |

---

## 4. 目标用户

**核心用户**：
- AI 学习者（初学者到中级）
- Claude Code / Cursor 等 AI 工具使用者
- 对 AI 应用开发感兴趣的开发者

**用户画像**：
- 年龄：20-40 岁
- 背景：软件工程师 / 学生 / 技术爱好者
- 需求：寻找 AI 入门教程、工具使用指南、案例参考

---

## 5. 用户访问网站最核心要完成的事

**场景 1**：访客想了解某个 AI 概念（如 token）→ 进入知识库 → 展开分类 → 选择文章 → 阅读正文

**场景 2**：访客想学习 Claude Code 使用 → 进入知识库 → 展开分类 → 选择文章 → 阅读教程

**场景 3**：访客想了解最新的 AI 技术文章 → 进入文章分享 → 阅读解读内容

---

## 6. 四个页面各自的目标

### 6.1 首页
- **目标**：快速了解站点定位 + 引导进入核心功能
- **用户路径**：访客 landed → 3秒内理解这是做什么的 → 点击进入知识库或文章分享

### 6.2 知识库
- **目标**：结构化展示 AI 知识体系，支持按分类浏览和文档式阅读
- **内容形态**：词条式内容（如解释什么是 token）、教程式内容（step by step）
- **布局模式**：三栏式文档布局（左侧分类导航 + 中间正文 + 右侧目录）
- **用户路径**：进入知识库 → 展开分类 → 选择文章 → 阅读正文

### 6.3 文章分享
- **目标**：展示经过整理和解读的外部文章
- **内容形态**：文章摘要 + 个人解读 + 原文链接
- **用户路径**：浏览文章列表 → 点击阅读解读 → 如需原文可跳转

### 6.4 关于我
- **目标**：建立作者可信度 + 提供联系方式
- **内容**：个人简介、技术背景、联系方式（邮箱、GitHub 等）

---

## 7. "知识库"和"文章分享"的边界区别

| 维度 | 知识库 | 文章分享 |
|------|--------|----------|
| **内容性质** | 原创整理的知识点、概念解释、教程 | 外部文章的解读、评论、摘要 |
| **内容形态** | 独立词条/词条集合（如"什么是token"） | 单篇解读文章 |
| **创作方式** | 完全原创，用自己的话解释 | 引用外部内容 + 个人解读 |
| **结构** | 一级分类体系，按主题组织 | 按文章发布倒序排列，可加标签 |
| **布局模式** | 三栏式文档布局（分类导航 + 正文 + 目录） | 列表页 + 详情页分离模式 |
| **更新频率** | 低（整理好了就稳定） | 中（看到好文章就分享） |

**简单理解**：知识库 = "我会的"，文章分享 = "我看到的"。

## 8. 网站的信息架构建议

```
/
├── 首页 (Home)
│   - 站点简介
│   - 核心入口（知识库 / 文章分享）
│   - 精选内容推荐
├── 知识库 (Knowledge) - 三栏布局
│   ├── 左侧：分类导航（可展开/收起）
│   ├── 中间：正文内容
│   └── 右侧：目录（TOC）
│   └── 分类结构：
│       ├── AI基础
│       │   ├── 什么是 token
│       │   └── ...
│       ├── Claude Code 使用教程
│       │   ├── 入门指南
│       │   └── ...
│       └── AI应用开发
├── 文章分享 (Articles) - 列表+详情模式
│   ├── [文章标题] - 解读
│   └── ...
└── 关于我 (About)
    - 个人简介
    - 联系方式
```

**导航建议**：
- 顶部固定导航栏：首页 | 知识库 | 文章分享 | 关于我
- 知识库页面：三栏布局（左侧分类导航 + 中间正文 + 右侧目录）
- 文章分享页面：列表页 + 详情页分离模式

## 16. 知识库结构决策

### 采用方案：三栏式文档布局

**布局结构**：
- **左侧**：一级分类导航，分类可展开/收起，展开后显示该分类下的文章列表
- **中间**：当前选中文章的正文（Markdown 渲染）
- **右侧**：当前文章的目录（TOC），基于文章内容自动生成

**交互方式**：
1. 左侧分类可以展开/收起
2. 展开后可以看到该分类下的文章列表
3. 点击文章后，中间区域切换为该文章正文
4. 右侧目录基于当前文章内容生成
5. 点击目录项可跳转到正文对应位置

**原因**：
1. **阅读体验更好**：文档式三栏布局更适合技术文档阅读，左侧快速导航，中间专注阅读，右侧目录定位
2. **用户路径更短**：一步到位，不需要从列表页跳转到详情页
3. **MVP 阶段内容少**：3 个一级分类，每个分类下的文章数量有限，三栏布局完全够用
4. **后续可扩展**：如果未来内容增多，可以轻松添加搜索或二级分类

**MVP 阶段的分类结构**：
```
AI基础
Claude Code 使用教程
AI应用开发
```
