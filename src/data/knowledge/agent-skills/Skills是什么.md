Agent Skills 是一种**轻量级、开放式的格式**，用于通过专门的知识和**工作流**来扩展 AI 代理的能力。

从本质上说，**一个 skill 就是一个包含 `SKILL.md` 文件的文件夹**。这个文件至少会包含元数据（最少要有名称和描述）以及操作说明，用来告诉代理如何完成某个特定任务。**技能还可以附带脚本、模板和参考资料。**

```txt
my-skill/
├── SKILL.md          # 必需项：指令 + 元数据。
├── scripts/          # 可选项：可执行代码。
├── references/       # 可选项：文档。
└── assets/           # 可选项：模板、资源。
```

# 1. skill 是如何工作的

Skills 使用 **渐进式披露（progressive disclosure）** 来更高效地管理上下文：

* **发现阶段（Discovery）**：在启动时，代理**只会加载每个可用 skill 的名称和描述**，仅保留足够的信息来**判断**它**何时可能相关**。
* **激活阶段（Activation）**：**当某个任务与某个 skill 的描述匹配时**，代理才会把完整的 `SKILL.md` 指令读入上下文。
* **执行阶段（Execution）**：代理按照这些指令执行任务，并在需要时选择性加载被引用的文件或运行 skill 附带的代码。

这种方式既能让代理保持高效轻量，又能在需要时按需获得更多上下文。



**一个例子来说明 Skills**：

假设你有一个 AI agent，它本来只是“普通助手”。

* 你问它：

  > 帮我写一个产品需求文档

  它当然也能写，但可能会：

  * 有时先写功能，有时先写页面
  * 有时漏掉用户场景
  * 有时漏掉验收标准
  * 每次输出风格都不一样

* 这时你就可以给它加一个 skill，比如叫：**prd-writer**

  这个 skill 的 `SKILL.md` 里可能写着：

  * 什么时候用：当用户要写 PRD、需求文档、功能说明时使用
  * 应该怎么做：
    * 先明确产品目标
    * 再列用户角色
    * 再写核心场景
    * 再写功能列表
    * 最后补验收标准和风险

* **加了这个 skill 之后会发生什么**

  以后 AI 再遇到：

  > 帮我写一个“个人 AI 知识博客”的需求文档

  它就不会随便发挥，而是会按这个 skill 的流程来：

  1. **先帮你定义目标**。例如：知识沉淀、公开分享、个人品牌展示
  2. **再识别用户角色**。例如：站点访客、站点维护者（你自己）
  3. **再拆核心页面**。例如：首页、知识库、文章分享、关于我
  4. **再写每页功能**。例如：知识库支持二级分类、Markdown 渲染、移动端适配
  5. **最后补验收标准**。例如：四个页面可访问、分类浏览正常、代码块渲染正常

  这样，AI 就像突然“学会了一套写 PRD 的专业方法”。

> Agent Skills 不是在“换一个更聪明的 AI”，而是在：
>
> **给原来的 AI 增加某一类事情的专业操作手册。**(就好像让 AI 突然学会了什么技能（工作流），后续再遇到类似的场景，直接使用相关的技能、一套包装好的工作流程来进行稳定、高效地工作)



# 2. SKILL.md 文件

每个 skill 都以一个 `SKILL.md` 文件开始，其中包含 **YAML frontmatter** 和 **Markdown 指令内容**：

```markdown
---
name: pdf-processing
description: 提取 PDF 文本、填写表单、合并文件。在处理 PDF 时使用。
---

# PDF 处理

## 什么时候使用这个技能
当用户需要处理 PDF 文件时，使用这个技能……

## 如何提取文本
1. 使用 pdfplumber 提取文本……

## 如何填写表单
……
```

以下 frontmatter 必须位于 `SKILL.md` 文件的顶部：

- `name`：一**个简短的标识符**
- `description`：**说明何时使用这个 skill**

Markdown 正文部分则**包含实际的说明内容**，在结构或内容上没有特定限制。



这种简单格式有几个关键优势：

- **自说明性强**：skill 的作者或使用者只要阅读一个 `SKILL.md`，就能明白它的作用，这使得 skill 很容易审查和改进。
- **可扩展性强**：skill 的复杂度可以从单纯的文字说明，扩展到可执行代码、资源文件和模板。
- **便于移植**：skill 本质上只是文件，因此很容易编辑、做版本管理和分享。
